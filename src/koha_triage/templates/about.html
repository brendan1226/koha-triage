About koha-triage

AI-powered triage for the Koha community Bugzilla

What is this?

koha-triage harvests bugs from bugs.koha-community.org and makes them searchable using semantic AI. Instead of keyword matching, you can describe a problem in plain language and find related bugs — even if they use different terminology.

Built by ByWater Solutions to help the Koha community triage, prioritize, and fix bugs faster.

Features

S
Semantic search

Describe a problem in your own words. koha-triage uses AI embeddings to find the most relevant bugs, regardless of exact wording.

V
AI verdicts

Each search result gets a Claude-generated verdict: is it a duplicate? Already fixed? Has a patch? So you know what action to take.

G
Bug grouping

Manually cluster related bugs into groups for coordinated triage — useful when multiple reports describe the same underlying issue.

R
AI recommendations

For any bug, generate a structured fix recommendation using Koha's coding guidelines and development handbook. Identifies affected files, complexity, and a test plan.

C
Code generation

AI generates targeted code fixes displayed as git-style diffs. Download as a .patch file ready for the git bz workflow.

P
Patch rebasing

For bugs with status "Patch doesn't apply," koha-triage fetches the stale patch from Bugzilla, rebases it against the current code, and credits the original author.

How it works

  1. Harvest — Bugs and comments are pulled from the Bugzilla REST API and stored locally.
  2. Embed — Each bug's summary and description are converted to a vector embedding for semantic search.
  3. Search — Your query is embedded and compared against all bugs using cosine similarity.
  4. Classify — Claude reads the top matches and assigns a verdict to each one.
  5. Recommend — For a specific bug, Claude analyzes it against Koha's coding guidelines and produces a structured fix plan.
  6. Generate — Claude writes the actual code fix, shown as a diff you can review and download as a patch.

Technology

Python, FastAPI, SQLite, fastembed (BAAI/bge-small-en-v1.5), and Claude by Anthropic. Source code available on GitHub.

All AI-generated code includes an Assisted-by trailer per Koha's AI contribution requirements. Human review is always required before submitting patches.

Koha patch workflow

Koha does not accept pull requests. The contribution workflow is:

  1. Find or file a bug on Bugzilla
  2. Generate a fix here (or write one yourself)
  3. Download the .patch file
  4. Apply locally: git am bug_XXXXX.patch
  5. Review, test, and adjust
  6. Attach to Bugzilla: git bz attach XXXXX HEAD
